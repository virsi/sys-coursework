# 2. Модульная структура программного обеспечения

Программа делится на резидентную и нерезидентную (инициализирующую) части. Резидентная часть реализует функционал данного программного обеспечения. Нерезидентная часть нужна для инициализации резидентной части, обработки параметров командной строки и выгрузки резидента по повторному запуску программы.

См. документ «Модульная структура программы».

# 5. Основные алгоритмы программного обеспечения

**Ход инициализации.** Устанавливается требуемый видеорежим для вывода текстовых сообщений во время работы резидента, вызывается обработчик параметров командной строки, затем, если программа запущена без параметров и резидент отсутствует в памяти, происходит установка резидента и удаление из ОП кода ниже метки `_initTSR`. Если задан флаг `/?`, выводится справка по работе с программой. Если же программа запущена без параметров и резидент уже находится в памяти, то выполняется выгрузка резидента через INT 2Fh с восстановлением старых обработчиков прерываний (требование группы ИУ5-42Б).

**Обработчик new\_int2Fh.** Вначале проверяется, равен ли регистр AH значению 0FFh. Если это так — это наша подфункция, и при AL=0 в AH заносится `'i'` как признак того, что резидент уже загружен в память. При AL=1 выполняется выгрузка резидента из памяти — восстанавливаются старые обработчики прерываний INT 09h, INT 1Ch, INT 2Fh, освобождается PSP-окружение и тело резидента.

**Обработчик new\_int9h.** Из порта 60h считывается скан-код нажатой клавиши. При нажатии функциональных клавиш F8, F9, F1, F2 меняют свои значения флаги соответствующих режимов (вывод подписи, курсив буквы «З», русификация клавиатуры, ограничение ввода русских букв), а также обновляется их индикация в правом нижнем углу экрана. Затем вызывается стандартный обработчик данного прерывания. Если введённый символ входит в множество игнорируемых (русские буквы при F2 активном), он удаляется из буфера клавиатуры. Если входит в множество translateFrom — заменяется на символ с тем же индексом из множества translateTo (при F1 активном).

**Обработчик new\_int1Ch.** В самом начале работы обработчика производится вызов старого обработчика прерывания INT 1Ch. В случае, если флаг signaturePrintingEnabled установлен в true, производится сравнение счётчика counter вызовов прерывания системой с числом printDelay\*1000/55 + 1, где printDelay — задержка в секундах (5). Если эти числа равны, далее производится печать информации об исполнителе курсовой работы на экран; иначе counter увеличивается на 1.

**Функция вывода подписи (printSignature).** Читается текущее положение курсора на экране и запоминается в стеке. Далее происходит выбор положения подписи на экране (верх, центр или низ) на основе значения переменной printPos. Для варианта № 5 printPos равно 0 — подпись выводится в верхнюю часть экрана. В каждом случае устанавливаются значения регистров DH и DL, хранящие информацию о строке и столбце соответственно. Затем эта информация используется для вывода построчно подписи, содержащей верхнюю рамку, три строки информации об исполнителе и нижнюю рамку. Восстанавливается положение курсора из стека. Вызывается функция changeFx для прекращения индикации вывода подписи.

**Функция вывода индикации (changeFx).** Читается текущее положение курсора и запоминается в стеке. Далее происходит перебор всех четырёх флагов состояний — signaturePrintingEnabled, cursiveEnabled, translateEnabled, ignoreEnabled. В случае, если флаг установлен в true, производится печать наименования функциональной клавиши соответствующего флага в правом нижнем углу экрана на зелёном фоне, иначе — на красном. Восстанавливается положение курсора из стека.

**Функция проверки командной строки (commandParamsParser).** В регистр SI помещается смещение 80h (адрес параметров командной строки в PSP). Читается количество символов в параметрах. Если их количество равно 0 — выходим. Иначе идёт цикл до тех пор, пока не будет прочитан символ возврата каретки. Если одним из параметров является `/?`, выводится справка по использованию программы и устанавливается флаг того, что загружать резидент не надо (notLoadTSR).

**Функция, меняющая изображение символа с курсива на обычное и наоборот (setCursive).** В стек сохраняются регистры AX и ES, в регистр ES загружается значение регистра CS. Далее, если флаг cursiveEnabled установлен в true, происходит сохранение текущего изображения изменяемого символа и последующая замена на новое. Номер изменяемого символа содержит переменная charToCursiveIndex (для варианта № 5 — 0x97 «З»). С помощью процедуры saveFont определяется адрес текущей таблицы символов. Затем, добавляя необходимое значение к регистру BP (16 \* charToCursiveIndex), находится адрес нужного символа и сохраняется 16 байт таблицы его изображения в переменную savedSymbol. После чего в регистр CX заносится 1, в DL устанавливается номер изменяемого символа, в BP перемещается адрес таблицы нового символа из переменной cursiveSymbol. Происходит вызов функции changeFont. Если флаг cursiveEnabled не установлен — происходит восстановление старого изображения символа.

См. документ «Блок-схема алгоритма программы».

# 7. Векторы прерываний программного обеспечения, переопределяемые в программе

В программе переопределяются три вектора прерываний:

1. **INT 09h** — обработка нажатия клавиш. Источник — аппаратное прерывание IRQ1 от клавиатурного контроллера 8042.
2. **INT 2Fh** — мультиплексор DOS. Используется для проверки наличия программы в памяти, а также для выгрузки резидентной части программы.
3. **INT 1Ch** — пользовательский таймерный обработчик. Используется для подсчёта времени, прошедшего после нажатия функциональной клавиши F8, для последующего вывода сообщения-подписи.

Сохранение старых обработчиков выполняется через INT 21h функция `35xx`, установка новых — через `25xx`. При выгрузке (по повторному запуску `tsr.com` без параметров) все три адреса восстанавливаются в исходное состояние.
