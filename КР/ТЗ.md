# 2. Назначение разработки

Разрабатываемое программное обеспечение представляет собой резидентную программу (TSR — Terminate and Stay Resident), функционирующую под управлением MS DOS версии 5.0 и выше или эмулятора DOSBox 0.74 и выше для современных версий Windows.

Программа предназначена для модификации поведения клавиатуры и экрана в режиме командной строки в учебных целях, а также для демонстрации работы механизмов перехвата прерываний, проверки повторности загрузки, корректной выгрузки и восстановления исходных обработчиков.

# 3. Требования к программе

## 3.1. Требования к функциональным характеристикам

Программа должна обеспечивать выполнение функций варианта № 5, приведённых в разделе 5.1.

## 3.2. Требования к надёжности

Программа должна корректно сохранять и восстанавливать векторы прерываний (INT 09h, INT 1Ch, INT 2Fh). При повторной загрузке программа не должна повторно занимать память. При выгрузке программа должна полностью освобождать как тело резидента, так и PSP-окружение.

## 3.3. Требования к составу и параметрам технических средств

1. IBM-совместимый компьютер с процессором 8086 и выше.
2. Не менее 3 Кбайт свободной оперативной памяти.
3. Дисплей с поддержкой текстового режима CGA/EGA/VGA 80×25.
4. Стандартная PC-клавиатура.

## 3.4. Требования к информационной и программной совместимости

Целевая операционная система — MS DOS 5.0 и выше. Допустимо исполнение в эмуляторе DOSBox 0.74 и выше под управлением Windows 7/10/11. Язык реализации — Assembler для процессора i8086. Средства разработки: Turbo Assembler 3.1, Turbo Link 5.1, Turbo Debugger 3.1.

# 4. Требования к программной документации

В составе документации должны быть разработаны: Техническое задание (настоящий документ), Описание применения, Техническое описание, Руководство пользователя, Руководство системного программиста, Программа и методика испытаний, Листинг программы, а также три листа формата А1 — модульная структура, схема взаимодействия с аппаратурой, блок-схемы алгоритмов. Оформление документации производится по ГОСТ серии 19 (Единая система программной документации).

# 5. Технические требования

## 5.1. Требования к выполняемым функциям

Программа должна обеспечивать выполнение следующих функций (функции варианта № 5 указаны первыми):

5.1.1. При нажатии клавиши **F8** программа через 5 секунд должна вывести в верхнюю часть экрана текстовое сообщение в рамке, содержащее: ФИО студента «Воробьев Е. А.», индекс учебной группы «ИУ5-42Б», номер варианта «Вариант № 5». Повторное нажатие F8 отключает режим автоматического вывода.

5.1.2. При нажатии клавиши **F9** программа должна модифицировать изображение русской буквы «З» на всём экране из обычного шрифта в курсивное начертание. Повторное нажатие F9 возвращает обычный шрифт.

5.1.3. При нажатии клавиши **F1** программа должна включить режим частичной русификации клавиатуры для пяти букв «ЕЖЗИК». Латинские клавиши, соответствующие этим буквам в раскладке ЙЦУКЕН (T, ;, P, B, R), должны выводить на экран соответствующие русские буквы. Повторное нажатие F1 отключает режим.

5.1.4. При нажатии клавиши **F2** программа должна ограничивать ввод с клавиатуры таким образом, чтобы русские буквы (строчные и прописные) не доходили до экрана. Латинские буквы, цифры и спецсимволы остаются доступны для ввода. Повторное нажатие F2 восстанавливает обычный режим ввода.

5.1.5. При запуске с ключом `/?` программа должна вывести в командную строку справку по использованию (БНФ, описание ключей и функций). При этом проверка повторности загрузки и загрузка резидента в память не производятся.

5.1.6. Программа должна обеспечивать загрузку в оперативную память с фиксацией в виде резидента при первом запуске без параметров.

5.1.7. При первой загрузке должны сохраняться адреса старых обработчиков прерываний INT 09h, INT 1Ch, INT 2Fh и устанавливаться новые обработчики.

5.1.8. Программа должна выполнять проверку повторности загрузки через INT 2Fh (подфункция `AH=FFh, AL=0`). Если резидент уже установлен, повторная загрузка не производится.

5.1.9. При повторном запуске программы без параметров (требование группы ИУ5-42Б) программа должна выгружать резидент из памяти, восстанавливать оригинальные обработчики прерываний и освобождать как PSP-окружение, так и тело резидента.

5.1.10. Программа должна выдавать сообщения: «Резидент загружен!» — при успешной первичной загрузке; «Резидент уже загружен» — при попытке повторной загрузки в нештатной ситуации; «Резидент выгружен» — при успешной выгрузке; «Недостаточно памяти» — при невозможности резидентного размещения; «Не удалось выгрузить резидент» — при неудачной выгрузке; «Ошибка параметров командной строки» — при неверных ключах.

5.1.11. В правом нижнем углу экрана программа должна отображать индикатор активных функций (F8, F9, F1, F2) — зелёный фон, если функция активна, красный — если выключена.

## 5.2. Требования к входным и выходным данным

Входные данные: аргументы командной строки (пусто или `/?`); скан-коды клавиш, получаемые через порт 60h и буфер клавиатуры по адресу 40h:1Eh.

Выходные данные: текстовый вывод в видеопамять B800h; сообщения о состоянии в стандартный поток вывода DOS.
