# Курсовая работа по дисциплине «Системное программирование»

> МГТУ им. Н. Э. Баумана · Кафедра ИУ-5 · 2 курс, 4 семестр, 2026 г.
> Студент: **Воробьев Е. А.**, группа **ИУ5-42Б**, вариант **№ 5**.

Резидентная программа (TSR — Terminate and Stay Resident) на языке Assembler i8086 для MS DOS / DOSBox. Демонстрирует механизмы перехвата прерываний, проверки повторности загрузки, корректной выгрузки из памяти и работы с шрифтом VGA.

В репозитории — полный комплект курсовой работы: исходный код, документация по ГОСТ ЕСПД (7 документов в `.docx`), три листа формата А1 в `.pdf` с рамкой и штампом по ГОСТ 2.104, а также автоматическая сборка через `make`.

## Параметры варианта № 5

| Клавиша | Функция | Параметры |
|---------|---------|-----------|
| **F8**  | Вывод подписи по таймеру | Через **5 секунд** в **верхней** части экрана выводится рамка: «Воробьев Е. А.» / «ИУ5-42Б» / «Вариант № 5» |
| **F9**  | Курсивное начертание | Русская буква **«З»** меняется на курсив; повтор — возврат |
| **F1**  | Частичная русификация | Латинские клавиши `T`, `;`, `P`, `B`, `R` → русские буквы **`ЕЖЗИК`** (по ЙЦУКЕН) |
| **F2**  | Ограничение ввода | Блокируются **русские буквы**, прочие символы — пропускаются |
| `tsr /?` | Справка | Выводит описание ключей, резидент не загружается |
| `tsr`   | Загрузка / выгрузка | Первый запуск — загрузка резидента; повторный — выгрузка (требование группы ИУ5-42Б) |

Перехватываются три вектора прерываний: **INT 09h** (клавиатура), **INT 1Ch** (таймер), **INT 2Fh** (мультиплексор DOS).

## Команды Makefile (выполняются из каталога `КР/`)

| Команда | Что делает |
|---------|------------|
| `make` или `make all` | Собирает **всё**: 7 `.docx` из `.md` + 3 листа А1 (`.pdf`) из SVG |
| `make docx` | Собирает только 7 `.docx`-документов (ТЗ, ОП, ТО, РП, РСП, ПМИ, ЛИТ) |
| `make листы` | Собирает только 3 листа А1 в `.pdf` через `rsvg-convert` |
| `make pdf` | Конвертирует все `.docx` → `.pdf` через LibreOffice (для сдачи) |
| `make zip` | Упаковывает комплект в `Воробьев_КР_СП_ИУ5-42Б_СП_2026.zip` — для отправки преподавателю |
| `make clean` | Удаляет собранные `.docx`, `.pdf`, листы А1 и `.zip` (исходники сохраняются) |
| `make clean-asm` | Удаляет артефакты компиляции `tsr.com`, `tsr.lst`, `tsr.obj`, `tsr.map` |
| `make help` | Показывает справку со списком всех целей |

### Типичные сценарии

**Отредактировал текст документа:**

```bash
cd КР

cd КР

```

## Команды runKR.bat (в DOSBox)

| Команда | Действие |
|---------|----------|
| `runKR`           | Полный цикл: компиляция + линковка + запуск (загрузка резидента) |
| `runKR build`     | Только сборка (`tasm /l tsr.asm` + `tlink /t /x tsr.obj`) |
| `runKR run`       | Только запуск `tsr.com` |
| `runKR mem`       | `mem /P` — карта памяти (для скриншотов ПМИ) |
| `runKR unload`    | Выгрузить резидент (повторный запуск `tsr.com`) |
| `runKR clean`     | Удалить `tsr.com`, `tsr.lst`, `tsr.obj`, `tsr.map` |
| `runKR help`      | Справка |

### Типичный сценарий для скриншотов ПМИ

```
runKR mem        → скриншот 1 (память до загрузки)
runKR build
runKR run        → «Резидент загружен!»
runKR mem        → скриншот 2 (виден блок TSR ~2,2 КБ)
                   # тестируем F8, F9, F1, F2
runKR unload     → «Резидент выгружен»
runKR mem        → скриншот 3 (память восстановлена)
```

## Как устроена сборка документации

1. Каждый документ имеет **markdown-исходник** (`ТЗ.md`, `ОП.md`, …) с титульным листом, содержанием с точечной заливкой и нумерованными разделами.
2. **Pandoc** конвертирует `.md` → `.docx`, подставляя стили из `reference.docx`:
   - Шрифт **Times New Roman 14 пт**;
   - Межстрочный интервал **1,5**;
   - Поля **30 / 15 / 20 / 20 мм** (ГОСТ ЕСПД);
   - Заголовки H1: жирный, прописные, по центру; H2: жирный, слева;
   - Стили `TOC1`/`TOC2` имеют `<w:tab leader="dot">` — содержание получается с точками и номерами страниц «как у Щепнова».
3. **Листы А1** написаны вручную в SVG (`листы/*.svg`) и конвертируются через `rsvg-convert` в PDF формата A1 (841 × 594 мм) с рамкой и штампом по ГОСТ 2.104.
